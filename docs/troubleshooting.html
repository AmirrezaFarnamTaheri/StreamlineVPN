üöÄ Installation Issues

Issue: "No module named streamline_vpn"

Symptoms:
  • ImportError when running the application
  • Python can't find the streamline_vpn module
  • Commands fail with module not found
Solutions:
# Ensure you're in the project directory cd /path/to/streamlinevpn # Install in development mode pip install -e . # Alternative: Add to Python path export PYTHONPATH="${PYTHONPATH}:$(pwd)/src" # Verify installation python -c "import streamline_vpn; print('Success!')"
üì¶ Dependency Conflicts

Issue: Package dependency conflicts

Symptoms:
  • pip install fails with conflict errors
  • Application crashes with import errors
  • Version compatibility issues
Solutions:
# Use virtual environment python -m venv streamline_env source streamline_env/bin/activate # Windows: streamline_env\Scripts\activate # Clean install pip install --upgrade pip pip install -r requirements.txt # Force reinstall if needed pip install --force-reinstall -r requirements.txt
üê≥ Docker Issues

Issue: Docker container fails to start

Symptoms:
  • Container exits immediately
  • Port binding errors
  • Volume mount issues
Solutions:
# Check container logs docker-compose logs streamline-api # Rebuild containers docker-compose down docker-compose build --no-cache docker-compose up -d # Check port conflicts netstat -tulpn | grep :8080 # Fix permissions (Linux/macOS) sudo chown -R $USER:$USER ./logs ./output
üîå API Connection Issues

Issue: API server not responding

Symptoms:
  • Frontend shows "API server unavailable"
  • HTTP 503 Service Unavailable errors
  • Connection timeouts
Solutions:
# Check if API server is running curl http://localhost:8080/health # Start API server manually python run_unified.py # Check server logs tail -f logs/streamline.log # Verify port binding sudo lsof -i :8080
üîê CORS Errors

Issue: Cross-Origin Request Blocked

Symptoms:
  • Browser console shows CORS errors
  • Frontend can't communicate with API
  • Preflight request failures
Solutions:
# Configure CORS environment variables export ALLOWED_ORIGINS='["http://localhost:3000","http://localhost:8000"]' export ALLOWED_METHODS='["GET","POST","PUT","DELETE","OPTIONS"]' export ALLOWED_HEADERS='["Content-Type","Authorization"]' # For development, allow all origins (not for production!) export ALLOWED_ORIGINS='["*"]'

üêå Slow Processing

Issue: VPN configuration processing is slow

Symptoms:
  • Pipeline takes >5 minutes to complete
  • High CPU/memory usage
  • Timeout errors during processing
Solutions:
# Reduce concurrent connections export VPN_CONCURRENT_LIMIT=25 # Default: 50 # Increase timeout export VPN_TIMEOUT=120 # Default: 60 seconds # Enable Redis caching for better performance export STREAMLINE_REDIS__NODES='[{"host":"localhost","port":"6379"}]' # Monitor resource usage htop # or Activity Monitor on macOS
üíæ Memory Issues

Issue: High memory usage or out of memory errors

Symptoms:
  • Application crashes with MemoryError
  • System becomes unresponsive
  • Docker container killed by OOM killer
Solutions:
# Limit concurrent processing export VPN_CONCURRENT_LIMIT=10 # Enable L3 SQLite cache to reduce memory usage export CACHE_L3_ENABLED=true # Increase Docker memory limit # In docker-compose.yml: services: streamline-api: deploy: resources: limits: memory: 2G
üîÑ Cache Issues

Issue: Redis cache not working

Symptoms:
  • Processing doesn't speed up after first run
  • Redis connection errors in logs
  • Cache hit rate is 0%
Solutions:
# Check Redis connection redis-cli ping # Test Redis connectivity from Python python -c "import redis; r=redis.Redis(); print(r.ping())" # Disable Redis if not needed (uses SQLite fallback) export STREAMLINE_REDIS__NODES='[]' # Clear cache if corrupted redis-cli FLUSHALL
üåê Source Connectivity

Issue: Unable to fetch from VPN sources

Symptoms:
  • Timeout errors when fetching sources
  • DNS resolution failures
  • SSL/TLS certificate errors
Solutions:
# Test source connectivity manually curl -I https://raw.githubusercontent.com/user/repo/main/configs.txt # Check DNS resolution nslookup raw.githubusercontent.com # Test with different timeout export VPN_TIMEOUT=180 # Bypass SSL verification (temporary fix) export PYTHONHTTPSVERIFY=0 # Not recommended for production
üîí Firewall Issues

Issue: Firewall blocking requests

Symptoms:
  • Connection refused errors
  • Requests hang indefinitely
  • Only some sources work
Solutions:
# Check firewall status (Ubuntu/CentOS) sudo ufw status # Ubuntu sudo firewall-cmd --list-all # CentOS # Open required ports sudo ufw allow 8080 # API port sudo ufw allow 8000 # Web port # Test with proxy if corporate firewall blocks direct access export HTTP_PROXY=http://proxy.company.com:8080 export HTTPS_PROXY=http://proxy.company.com:8080
