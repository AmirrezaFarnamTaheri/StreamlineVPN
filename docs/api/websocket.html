Endpoint

WebSocket URL: /ws (same host/port as API). The advanced client builds this automatically:

const ws = API.connectWebSocket('/ws');

Connection Lifecycle

  • On connect, server replies with a JSON message: {"type":"connection","status":"connected","client_id":"..."}
  • Client may send health pings: {"type":"ping"}
  • Server replies: {"type":"pong"}
  • Unrecognized messages echo back: {"type":"echo","received":{...}}

Message Shapes

// Client → Server (Ping) {"type": "ping"} // Server → Client (Pong) {"type": "pong"}
// Client → Server (Custom) {"type": "custom", "text": "hello"} // Server → Client (Echo) {"type": "echo", "received": {"type":"custom","text":"hello"}}

JavaScript Example

const ws = API.connectWebSocket('/ws'); ws.addEventListener('open', () => ws.send(JSON.stringify({ type: 'ping' }))); ws.addEventListener('message', (ev) => console.log('WS message', ev.data)); ws.addEventListener('close', (ev) => console.log('closed', ev.code, ev.reason));
Tip: Use WS Test to try messages quickly.
